# Revise Paper Card Workflow

1. Confirm the paper already exists in `canonical/papers/`.
2. Classify each user supplement as `replace`, `append`, `tighten`, or `verify`.
3. Edit only the targeted paper-card fields.
4. Preserve stable IDs and untouched sections.
5. If a rating changes, update both score and note.
6. Sync canonical markdown back into SQLite.
7. Re-export website data when the paper is publicly displayed.
